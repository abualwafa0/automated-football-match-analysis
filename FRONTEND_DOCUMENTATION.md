# التوثيق الفني لواجهة المستخدم (Frontend Web App)

يحتوي هذا الملف على شرح تفصيلي لكود واجهة الويب الخاصة بمشروع تحليل مباريات كرة القدم بالذكاء الاصطناعي، بما يشمل التقنيات المستخدمة، آلية العمل، وكيفية الربط مع خادم الذكاء الاصطناعي (Backend).

---

## 1. التقنيات ولغات البرمجة المستخدمة
تم بناء الواجهة باستخدام أحدث تقنيات الويب لضمان السرعة، التفاعلية، والشكل الجمالي الحديث:
- **React.js**: مكتبة الواجهة الأساسية (بناء المكونات التفاعلية UI Components).
- **TypeScript (`.ts`, `.tsx`)**: لغة البرمجة الأساسية (وهي نسخة مطورة من JavaScript تضمن خلو الكود من الأخطاء عبر تحديد أنواع البيانات `Typing`).
- **Vite**: أداة بناء وتشغيل خادم التطوير السريع جداً (بديل لـ Webpack/CRA).
- **Tailwind CSS**: إطار عمل التصميم لتنسيق العناصر والألوان بشكل مباشر وسريع.
- **Framer Motion**: مكتبة الحركات التفاعلية (Animations) التي تعطي تأثيرات الظهور السلسة للبطاقات والأزرار.
- **Lucide-React**: مكتبة الأيقونات المتجهات (SVG) المستخدمة في واجهة المستخدم.

---

## 2. بنية الكود (كيف يعمل النظام)
ينقسم كود الويب في مجلد `src` إلى عدة أجزاء رئيسية:
1. **`App.tsx` (المحرك الرئيسي):** هو المكون الأساسي الذي يدير حالة التطبيق بالكامل (هل المستخدم في الشاشة الرئيسية أم في لوحة التحكم؟ وما هي البيانات المستلمة؟).
2. **`components/` (المكونات):**
   - `LandingPage.tsx`: صفحة البداية التي تحتوي على التصميم الزجاجي وزر "ابدأ التحليل" وصور فريق العمل.
   - `DashboardHeader.tsx`: الشريط العلوي الذي يعرض أزرار تغيير اللغة والمحاكي وحالة الاتصال بالسيرفر.
   - `VideoSection.tsx`: قسم رفع الفيديو ومشغّل الفيديو الذي يتواصل مع السيرفر.
   - `PlayerStats.tsx` & `TeamStats.tsx`: جداول وبطاقات عرض الإحصائيات والأرقام بعد تحليلها.
3. **`contexts/LanguageContext.tsx`:** ملف لإدارة اللغات (عربي/إنجليزي) بشكل ديناميكي بحيث تتغير النصوص واتجاه الصفحة (RTL / LTR) بضغطة زر.
4. **`locales/translations.ts`:** قاموس الكلمات لجميع اللغات المدعومة.

---

## 3. كيف تم ربط الويب مع موديل الذكاء الاصطناعي (الـ Backend)
تتم عملية الربط والتواصل عبر بروتوكول HTTP (واجهة برمجة التطبيقات API). والمسؤول الأكبر عن هذه العملية هو ملف **`VideoSection.tsx`**:

1. **إرسال الفيديو (Upload & Request):**
   - عندما يقوم المستخدم بسحب وإفلات فيديو المباراة، يقوم الكود بوضعه داخل حاوية بيانات من نوع `FormData`.
   - يتم إرسال طلب (`POST`) إلى رابط السيرفر (FastAPI) المتمثل غالباً في `http://127.0.0.1:8000/`.
2. **المعالجة والانتظار (Processing):**
   - يظهر الويب شريط تحميل متفاعل (يقرأ البيانات والحالة) بينما يقوم سيرفر البايثون بتشغيل موديل الـ YOLO وتتبع اللاعبين واستخراج الخرائط الحرارية.
3. **استقبال النتائج (Response Handling):**
   - بعد انتهاء السيرفر، يُرجع استجابة بصيغة `JSON` تحتوي على إحصائيات المباراة (الاستحواذ، التمريرات، المسافات، إلخ) بالإضافة إلى مُعرّف الفيديو (`video_id`).
4. **عرض الفيديو المحلل (Playback):**
   - يقوم الويب بطلب الفيديو المعالج من مسار مخصص في السيرفر مثل `/video/{video_id}` ويقوم بتشغيله على الشاشة مع البيانات الناتجة التي تتوزع تلقائياً على المكونات (المخططات البيانية والجداول).

*(ملاحظة: يحتوي النظام أيضاً على وضع محاكي Simulator مدمج لاختبار الواجهة بدون الحاجة لتشغيل سيرفر الذكاء الاصطناعي الثقيل).*

---

## 4. كيف نقوم بتشغيله كسيرفر؟
لتشغيل واجهة الويب على جهازك أو كسيرفر محلي:

1. نفتح موجه الأوامر داخل مجلد الواجهة `football-web-ai-analytics`.
2. ننفذ أمر التثبيت (لتحميل مكتبات React وغيرها) ويتم هذا مرة واحدة فقط:
   ```bash
   npm install
   ```
3. ننفذ أمر تشغيل السيرفر المحلي:
   ```bash
   npm run dev
   ```
4. يعمل أداة `Vite` كخادم ويب سريع (Web Server)، ويقوم بتجميع كود الـ TypeScript وتحويله لكود مفهوم للمتصفحات (HTML/JS/CSS).
5. يعطيك في النهاية رابطاً (مثل `http://localhost:5173`)، وبمجرد فتحه في المتصفح يبدأ التطبيق بالعمل ويتصل بالواجهة الخلفية.
